\section{Kurzbeschreibung}
Diese Diplomarbeit befasst sich mit der Konzeption und Implementierung eines Zeitmesssystems für die Zero Emission Challenge (ZEC), einen Kartwettbewerb der HTL Weiz. \
Ausgangspunkt war eine bestehende Lösung, die sich als schwer wartbar, kaum erweiterbar und mit einem aufwendigen Setup-Prozess verbunden erwies, weshalb eine vollständige Neuentwicklung des Systems beschlossen wurde. \
Ziel war die Entwicklung einer Softwarelösung, die den gesamten Ablauf von der Zeitmessung über die Punkteauswertung bis hin zur Anzeige der Ergebnisse abdeckt und dabei eine hohe Erweiterbarkeit sowie eine einfache Inbetriebnahme gewährleistet. \
Die implementierten Kernkomponenten umfassen einen Backend-Server zur Verarbeitung und Verwaltung der Daten (realisiert mit Python und FastAPI), eine Webanwendung zur Erfassung von Zeitstempeln, welche diese per MQTT-Protokoll von Lichtschranken empfängt und an den Server weiterleitet, sowie eine weitere Webanwendung zur Darstellung der Ergebnisse und zur Verwaltung von Teams und Benutzern. \
Die Authentifizierung und rollenbasierte Zugriffskontrolle werden über Keycloak realisiert, während die Datenhaltung in einer PostgreSQL-Datenbank erfolgt. \
Das System basiert auf einer Microservice-Architektur und kann in einem Kubernetes-Cluster betrieben werden, wodurch Deployment und Hosting wesentlich vereinfacht werden. \
Zur Sicherstellung der Softwarequalität wurde der Entwicklungsprozess durch automatisierte CI/CD-Pipelines unterstützt, welche das Testen, Bauen und Veröffentlichen der Container-Images übernehmen.
